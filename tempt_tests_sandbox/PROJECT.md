# PROJECT.md: ボートレース予測モデル開発プロジェクト

## 1. プロジェクト概要

機械学習を用いてボートレースのレース結果を予測し、回収率の最大化、または的中精度の向上を目指すプロジェクト。

* **対象ドメイン**: ボートレース（競艇）
* **主要データソース**: ボートレース公式Webサイト（過去のレース結果、番組表、選手データ等）

---

## 2. システムアーキテクチャ（暫定）

1. **Data Ingestion**: 公式サイトからのスクレイピングおよびCSV等のパース
2. **Data Preprocessing**: 特徴量エンジニアリング、欠損値処理、正規化
3. **Model Training**: 学習アルゴリズムの実装とハイパーパラメータチューニング
4. **Evaluation**: バックテストによる回収率・的中率の検証
5. **Inference**: 未来のレースに対する予測実行

---

## 3. 決定するべき事項（抜け漏れチェックリスト）

プロジェクトを具体化するために、以下の項目を定義・決定する必要があります。

### 3.1 目的の定義（Objective）

* [ ] **予測のゴール**:
* 3連単の的中率を最大化するのか？
* 的中率が低くても「回収率（ROI）」を100%超えさせるのか？


* [ ] **ターゲット変数の設定**:
* 1着を当てる（分類問題）
* 各艇の入線順位を当てる（ランキング学習）
* タイムを予測する（回帰問題）



### 3.2 扱うアルゴリズムの選定

* [ ] **候補アルゴリズム**:
* 勾配ブースティング決定木（XGBoost, LightGBM, CatBoostなど）
* ニューラルネットワーク（PyTorch, TensorFlow）
* アンサンブル学習の構成



### 3.3 特徴量エンジニアリングの範囲

* [ ] **静的データ**: 選手級別、勝率、複勝率、モーター番号等
* [ ] **動的データ**: 当日の展示タイム、風速、波高、チルト角等（直前情報の扱い）
* [ ] **独自指標**: 独自に算出する期待値、ランク付けロジックの有無

### 3.4 評価指標（Evaluation Metrics）

* [ ] **機械学習指標**: LogLoss, Accuracy, F1-score等
* [ ] **ドメイン指標**: 回収率（期待値ベース）、的中率、最大ドローダウン

### 3.5 運用・実行環境

* [ ] **開発環境**: Pythonバージョン、ライブラリ構成（Poetry/Docker等）
* [ ] **ハードウェア**: ローカルMacBook、クラウドGPU、GWS等

---

## 4. ロードマップ

1. **Phase 1**: データ収集基盤の構築（スクレイピングスクリプト作成）
2. **Phase 2**: ベースラインモデルの作成（最小限の特徴量での学習）
3. **Phase 3**: 特徴量エンジニアリングとアルゴリズム比較
4. **Phase 4**: バックテストシステムの構築とシミュレーション
5. **Phase 5**: 実運用（推論）およびモデルの改善
